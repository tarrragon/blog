放量分配回答的問題是：主力供應商量產爬坡失敗時，產線會停多久。答案由兩個參數決定——其餘供應商短期能多吸收多少產能、以及 [安全庫存](/business/procurement-planning/cards/safety-stock/) 能撐多長的補救時間。主力家的放量上限是從這兩個參數反推出來的，任何具體比例（例如「不超過七成」）都是特定參數組合下的解，換了參數要重算。

## 為什麼驗證通過不等於量產可信

概念驗證（POC）跟 [NPI](/business/procurement-planning/cards/npi/) 試產回答的是「技術上做得出來」，量產爬坡回答的是「大量、連續、穩定地做得出來」——兩者是不同的統計問題。小量試產看到的是良率分佈的中心，量產才會暴露分佈的尾巴：製程窗口偏窄的批次性不良、換班與換線的變異、材料批次的相容性、產能拉高後的設備稼動瓶頸。一家供應商 POC 跟 NPI 全數通過、量產爬坡才失靈，是這個統計結構的正常展現，所以放量設計必須把「爬坡失敗」當成有真實機率的事件，而不是驗證通過後就歸零的殘餘風險。

## 上限的反推邏輯

用一組示意參數走一遍推導（比例為示意、非通用值）：

主力家拿七成、替補家合計拿三成。主力爬坡失敗、供貨歸零時，總供給只剩三成。替補家短期能擴多少？assembly 型（封裝、組裝這類以人力與班次為主）的產能靠單改雙班、假日加班，短期可以接近倍增——三成變六成，對照總需求還缺四成；wafer 型（晶圓製造這類長週期資本投資）的新增產能短期擴不動（已投片的在製品可用 hot lot 插隊壓縮週期，但那是加速、不是增量），缺口就是整整七成。倍增還有兩個前提：瓶頸要在班次而不在模具穴數這類硬體上限；受規管產業的核准綁定廠址與產線，加班可以、換線加產能要走變更通知與重新認證，短期彈性打折。缺口由安全庫存吸收：庫存能撐的天數，就是留給替補繼續擴產、或主力恢復的補救時間——缺口越大，同樣的庫存撐的天數越短，這是上限要壓低的直接理由。

這個推導把「七成」還原成機制：它成立的前提是替補持有三成且產能型態允許短期倍增。前提變了，上限跟著變——

| 情境參數                         | 對主力上限的影響                     |
| -------------------------------- | ------------------------------------ |
| 替補是 assembly 型、可短期倍增   | 上限可以放高，缺口能快速補上         |
| 替補是 wafer 型、短期擴不動      | 上限要壓低，或用更厚的庫存換         |
| 安全庫存厚（覆蓋週期長）         | 補救時間長，上限可以放高             |
| 替補家只有一家、自身也有失效機率 | 上限要再收，兩家同時出狀況的機率非零 |

每一列都是同一條式子的不同代入。判讀時的操作是：先問「主力失效時、我能容忍的斷供時間是幾天」（由庫存決定），再問「這段時間內替補能爬到多少」（由產能型態決定），兩個答案夾出主力的上限。

## 放量跟著驗證證據走

分配比例沿著驗證階段遞增，每一階段的加量對應新取得的證據——信任是用資料買的，不是用時間熬的。

| 階段           | 這階段產生什麼證據                                                     | 放量對應                     |
| -------------- | ---------------------------------------------------------------------- | ---------------------------- |
| POC            | 概念與規格可行                                                         | 小量試作、還沒進入分配       |
| NPI            | 試產良率、跟研發對齊的規格                                             | 有限投量、驗證量產前提       |
| 正式核准       | [PPAP / 首件](/business/procurement-planning/cards/ppap/) / 信賴性通過 | 進入可分配名單               |
| 量產（MP）初期 | 爬坡段的實際良率與交期                                                 | 按反推上限放量、保留替補空間 |
| MP 穩定        | 連續量產的統計表現                                                     | 依表現重分配、仍不觸頂上限   |

表格之外要說清楚一件事：MP 穩定後的重分配，調的是各家之間的相對份額，上限本身仍由替補產能的反推式管住。一家表現再好，把量全押給它等於把保單退掉——[第二來源的經濟學](/business/procurement-planning/multi-source-supplier-strategy/) 算過的保費就白繳了。

## 失效條件與邊界

**獨家料沒有分配問題**：技術上只有一家做得出來的料，分配框架不適用，風險處理走庫存厚度與產能合約（見 [原廠與代理商規則](/business/procurement-planning/vendor-lifecycle-rules/)）。

**新產品初期的量太小**：總量小到拆給兩家會讓每家都低於經濟批量（單位成本與品質風險開始明顯上升的最小訂單量）時，硬拆會推高兩邊的單價與品質風險，此時常見的過渡是單源量產、雙源認證——第二家保持認證有效但不投量，等量爬起來再啟動分配。

**上限是天花板、不是目標**：反推出的上限是「不能超過」的邊界，不是「應該達到」的配額。商務上有時會刻意壓低主力份額來維持競爭張力，那是議價籌碼的考量，跟這裡的風險反推疊加使用。
