這個模組用商業分析的語言拆解電子業採購 planning：推導「這些做法為什麼成立」的機制——成本結構、誘因、市場機制——讓讀者能把判準帶回自己的情境重算，而操作清單交給機制自己長出來。素材來自採購從業者的實戰經驗，模組的工作是把經驗裡的判準還原成可推導的框架：為什麼絕不斷料是理性的、養第二供應商的保費怎麼對比期望損失、缺貨時供應商按什麼順序配貨、原廠的訂單規則在管理誰的風險。

推導的起點是一個成本不對稱：斷料的損失比呆料的損失大數個量級——一顆 0.5 美元的電容能停下一條日產百萬美元的線。模組的每一篇都從這個不對稱往外展開：它推導出五條工作線的存在理由、第二來源的保險定價、漲價談判的籌碼力量、備料的覆蓋邏輯、呆料的停損判準、急救選單的排序、以及關係投資在配給中的價值。文中的算式與數字為量級示意，用來呈現推導結構，不是任何實際案例的數據。模組以品牌廠自主採購為預設視角；代工（EMS / ODM）情境的客戶端變數——客戶控制的 AVL、客供料、客戶產線的停線求償——只在相關段落點到，未展開成獨立主題。

## 這個模組解什麼問題

技術背景的人看採購工作，容易停在「下單、催貨、比價」的表面。這個模組回答的是決策層的問題：一顆料該備多少、備在哪一層，用什麼推導；一家供應商該不該養、養的成本值不值；料快斷時每個急救選項的代價是什麼、依什麼排序；一顆料的規則（可不可取消、價格誰控、還能活多久）怎麼轉成下單策略。讀完帶走的目標是判斷框架——參數換了（產業、規模、料的性質）能自己重算，而不是一套要背的行規。

## 建議讀法

| 路線         | 適合讀者                             | 建議順序                                                                                                                                          |
| ------------ | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| 完整推導     | 想從機制理解採購決策的讀者           | 成本結構 → 第二來源經濟學 → 漲價應對 → 放量分配 → 分層備料 → 呆料處置 → 追料 → 配給 → 原廠規則 → 生命週期退場 → LTB 買斷 → 職場筆記（經驗層收尾） |
| 備料決策     | 要負責備料量與庫存高度的 planner     | 分層備料 → 成本結構的失效條件段 → 原廠規則 → LTB 買斷 → 呆料處置                                                                                  |
| 供應商佈局   | 要規劃雙源與放量的 sourcer           | 第二來源經濟學 → 量產放量分配 → 漲價應對 → 配給                                                                                                   |
| 危機處理     | 手上正在追一顆快斷的料               | 追料 → 配給的順位段 → 分層備料的安全庫存段                                                                                                        |
| 理解採購約束 | RD 或 PM，想懂為什麼快好便宜不能全要 | 配給 → 原廠規則 → 第二來源經濟學的等待期段                                                                                                        |
| 檢核決策品質 | 想質詢備料與供應商決策的主管         | 各篇失效條件段當質詢清單：成本結構的反轉段 → 放量分配的邊界段 → 分層備料的失效段 → 漲價應對的失效表                                               |
