# 模組零：設計哲學（序章）

在深入學習 Python 技術細節之前，讓我們先建立一個統一的視角：**所有程式碼設計原則的最終目的都是「降低閱讀者的認知負擔」**。

## 為什麼需要這個序章？

你可能聽過很多設計原則：DRY、SOLID、Clean Code、重構技巧...但這些原則為什麼存在？它們的共同目標是什麼？

本模組將回答這個根本問題，並提供一個統一的框架來理解所有設計決策。

## 核心論點

```
程式碼的品質 ≠ 優美
程式碼的品質 = 易讀
```

無法讀懂的程式碼沒人會讀，更不用說重構或除錯。所有的設計原則都是為了讓程式碼更容易被人類理解。

## 學習目標

完成本模組後，你將能夠：

1. 用「降低認知負擔」的統一視角理解所有設計原則
2. 在命名時考慮閱讀者的認知負擔
3. 用認知負擔來評估設計決策的好壞
4. 理解為什麼某些程式碼「感覺不對」

## 與其他模組的關係

本模組是後續所有模組的理論基礎：

- **模組一到五**：學習技術細節時，思考這些技術如何降低認知負擔
