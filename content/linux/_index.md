這個分類收攏所有 Linux 相關的內容：把一台機器從空的裝起來、出問題時怎麼判、每個情境有哪些工具可用、以及怎麼把整套工作環境用 code 管理起來。Linux 的特點是靈活——同一件事往往有很多種工具、桌面環境也有多種版本與各自的擴充，所以「知道有哪些選擇、該用哪個」本身就是重要的能力，這個分類把這些一起整理。

## 子分類

| 子分類                       | 內容                                                                          |
| ---------------------------- | ----------------------------------------------------------------------------- |
| [安裝與機器初始化](install/) | 從 ISO 裝好 OS、判讀安裝選項、驗證最小系統、外部連入與 bootstrap 前置         |
| [除錯與診斷](debug/)         | 遠端 / 本地除錯的判讀紀律：讀權威狀態不靠肉眼、按症狀分流到對的檢查與工具     |
| [工具選單](tools/)           | 各情境推薦用哪些工具：CLI 環境、圖形桌面、遠端；同一件事的多種選擇怎麼挑      |
| [Dotfile 管理](dotfile/)     | 用 code 管理工作環境的完整方法論：從心智模型、shell、終端機、桌面到同步與團隊 |

## 建議的路線

- **拿到一台新機器**：[安裝與機器初始化](install/) → 裝好後用 [Dotfile 管理](dotfile/) 把環境一鍵還原。
- **系統出問題**：[除錯與診斷](debug/)，從診斷心法建立判讀紀律再依症狀分流。
- **想升級工具流**：[工具選單](tools/)，看你天天用的 CLI / 桌面 / 遠端工具有沒有更好的選擇。
