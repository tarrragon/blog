# 模組 3.5：進階設計模式

本模組將元編程的「機制理解」轉化為「架構設計能力」，學習如何建立型別安全、錯誤處理完善、可擴展的 Python 系統。

## 為什麼需要這個模組？

在入門系列中，我們學習了：

- 型別提示的基礎用法
- 異常處理的基本策略
- `with` 語句的使用
- 單例與快取模式

這些知識足以應付日常開發，但當你要設計：

- **可重用的 Repository 層**：需要泛型進階技巧
- **大型專案的錯誤處理架構**：需要異常層級設計
- **資源管理複雜的系統**：需要進階上下文管理
- **支援擴展的框架**：需要插件系統設計

你就需要本模組的進階設計模式。

## 模組定位

```text
模組二 元編程 → 模組 3.5 設計模式 → 模組三 CPython
   (機制)           (應用)            (底層)
```

元編程教你「如何建立抽象機制」，本模組教你「如何應用這些機制設計系統」。
