這個系列研究一件事：怎麼把 AI 或文件的輸出塑造成「不同神經類型的大腦都能實際拿去用」的形狀。同一份技術上正確的答案，對一個 ADHD 讀者可能因為把重點埋在第三段而無法啟動，對一個自閉讀者可能因為用了暗示語氣而無法確定該不該做，對一個有需求迴避特質的人可能因為開頭一句「你必須」就整個關機。輸出的品質不只取決於內容對不對，也取決於它有沒有配合讀者的認知運作方式。

系列的起點是三個真實存在的 Claude skill。它們各自針對一種神經多樣性特性，把「該怎麼輸出」寫成明確規則。本系列先逐一拆解這三個 skill 在解什麼問題、機制為何有效，再抽出跨三者共用的輸出設計方法論，最後把三種特性整合成一個可組合的 skill。

## 為什麼放在一起談

三種特性對「好輸出」的定義並不一致，甚至彼此衝突。ADHD 導向要求開頭就是一個可立即執行的動作（範例形式是祈使命令）；需求迴避導向卻要求把所有「你必須 / 你應該」拿掉。把三者放在同一個系列，是為了看清一件事：它們共用一層底層邏輯（降低提取成本、外化不該由讀者記住的狀態、誠實不操弄），差異只發生在上層的轉換方式。認清哪些是共用底層、哪些是可切換的上層，是整合成單一 skill 的前提。

| 特性            | 針對的認知現實                     | 輸出的核心轉換                            |
| --------------- | ---------------------------------- | ----------------------------------------- |
| ADHD            | 工作記憶小、啟動最難、多巴胺稀缺   | 壓縮 + 動作直給 + 外化狀態 + 讓進度可見   |
| 自閉（ASD）     | 字面理解、曖昧耗能、變動令人迷失   | 消歧義 + 明確結構 + 標優先級 + 變動前預告 |
| 需求迴避（PDA） | 「被要求」本身觸發焦慮、需要自主權 | 去命令化重構 + 給真實選擇 + 歸還控制權    |

三種轉換不在同一層。ADHD 與 ASD 改的是**輸出的格式**（同一件事怎麼排版、怎麼措辭）；PDA 改的是**任務的框架**（這件事被呈現成命令還是探索）。這個層次差異是整個方法論的樞紐，在方法論篇會展開。

## 定位與邊界

本系列談的是**輸出設計**，不是臨床指南。三種特性的描述取自對應 skill 與公開來源，用來推導「輸出該長怎樣」，不用來做診斷或治療建議。本系列刻意只取「輸出摩擦」這一個切面，上面的對照表因此只列每種特性在哪裡卡——這是取景的選擇，不是主張這些特性只有困難面。其中需求迴避（PDA）跟 ADHD、自閉不同，它不是正式診斷、作為獨立建構在研究社群內仍有爭議；本系列引用它是為了推導輸出原則，不預設它是已確立的病因。神經多樣性是光譜，同一個標籤下的個體偏好差異很大——本系列產出的規則是預設值，不是規範每個人都適用的處方。實際使用時，讀者本人的回饋永遠優先於任何預設規則。

系列面向兩種讀者：想為神經多樣性的自己或親友調整 AI 輸出的人，以及想理解「輸出如何配合認知運作」這個設計問題的人。前者可直接跳到對應特性的分析篇與最終 skill；後者建議依序讀完三篇分析再進方法論。

本系列可以看成 [compositional-writing](/skills/compositional-writing/) 的延伸。後者為通用讀者的認知負荷與 token 成本塑形輸出；本系列把同一個「為認知運作塑形輸出」的方法，推到特定神經類型上。兩者共用的底層原則在方法論篇的 base layer 一節對照。
