格式標準的選型是一道組織成本的選擇題：採一份現成的公開標準、還是自建規範加上治理機制。這道題不是比技術能力 —— 多數格式標準能做的事自建規範也做得到 —— 而是比「消除格式爭論的成本」跟「綁定一份外部標準的形狀」哪個划算。本目錄分兩層看：response 格式（你的 JSON 回應長什麼樣）採不採現成標準、描述格式（你怎麼描述 API 的形狀）選哪一個。兩篇共用一條判準：一個標準會不會活、由生態採用決定、不由標準機構的背書決定。中性的治理判準（採標準 vs 自建規範）見 [11.10 API 規範治理](/backend/11-api-design/api-governance/)。
