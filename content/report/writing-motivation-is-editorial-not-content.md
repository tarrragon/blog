## 論述基礎與限制

本卡的論述基於八方雲集案例分析的引言第二次修訂。第一次修訂去掉了創辦故事（見 [#229](../analysis-opening-positions-question-not-narrative/)），第二版改為解釋寫作動機：「揚秦文章引用了 36% 但八方缺乏獨立判讀來驗證這個數字」。這段被判定為「我們的寫作原因、讀者不會在乎」而再次改寫。限制：觀察來自單一篇修訂，但原則適用於所有以系列形式產出的分析或教學文章。

---

## 核心原則

**文章引言回答「這篇分析什麼、核心問題是什麼」，不回答「為什麼有這篇文章」。** 寫作動機（「A 文章引用了 B 的數字但缺獨立佐證」「系列的覆蓋有這個缺口」「前一篇沒分析到這個面向」）是編輯團隊的生產決策，屬於 commit message 和編輯筆記的層級。讀者不追蹤文章系列的覆蓋地圖，「為什麼作者決定寫這篇」對讀者沒有資訊量。

兩者的區分：

| 層級   | 內容                                                                | 屬於                      |
| ------ | ------------------------------------------------------------------- | ------------------------- |
| 讀者層 | 這篇分析八方雲集的中央工廠模式，核心問題是毛利率差距 9pp 的結構原因 | 正文引言                  |
| 編輯層 | 揚秦文章引用了 36% 但缺獨立佐證，需要補一篇八方的財報蒐集和經營判讀 | commit message / 編輯筆記 |

---

## 反模式：把編輯缺口當引言

「前一篇引用了 X 但沒有獨立分析」這類句子有一個隱含假設：讀者知道「前一篇」是什麼、在乎系列的完整性、認為補缺口是閱讀本篇的理由。這三個假設對多數讀者不成立——讀者可能從搜尋直接跳到本篇，不知道「前一篇」的存在，也不在意系列規劃。

跨篇引用本身沒問題（「跟揚秦的模式差異是...」提供分析上下文），但引用的功能應該是**建立分析問題的上下文**，不是**解釋文章存在的理由**。

## 判讀徵兆

| 訊號                                                            | 該做的事                                  |
| --------------------------------------------------------------- | ----------------------------------------- |
| 引言出現「A 文章引用了 / 提到了」+ 「但缺 / 沒有 / 不足」的句型 | 刪除動機段，直接從分析問題定位開始        |
| 引言在解釋「為什麼需要這篇」                                    | 這是 commit message 的內容，移出正文      |
| 跨篇引用的功能是「交代寫作背景」而非「建立分析上下文」          | 改寫引用，讓它服務分析問題而非系列管理    |
| 讀者需要先讀過前一篇才能理解本篇引言                            | 引言的跨篇依賴太強 — 補足本篇自包含的定位 |
