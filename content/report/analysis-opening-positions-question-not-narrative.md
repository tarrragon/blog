## 論述基礎與限制

本卡的論述基於一次商業分析文章（八方雲集案例分析）的引言修訂。第一版用創辦人故事敘事（負債兩千萬、天母四坪水餃攤、第一天賣出七千顆），被判定為「一般讀者向的描述、非商務性的故事」而改寫。限制：觀察來自單一篇修訂，但原則適用於所有分析型文章的開頭功能。

---

## 核心原則

**分析文章的開頭承擔的功能是定位分析問題——讓讀者在第一段就知道這篇分析什麼、核心問題是什麼、為什麼這個結構值得拆解。** 創辦人時間軸敘事（何時創立、怎麼起家、規模怎麼長大）承擔的是另一個功能：建立品牌認同或情感投入。兩個功能服務不同讀者——前者服務來做結構判讀的人，後者服務來認識品牌的人。

分析型開頭的三個要素：

1. **定位**：這間公司是什麼（產業、規模、核心數字）
2. **結構差異**：跟同業或參考點的關鍵差異在哪裡
3. **分析問題**：本篇要回答什麼問題

創辦敘事的資訊（創立年份、創辦人、起步過程）如果跟分析問題有因果關係就保留在正文對應段落，沒有因果關係就省略。「1998 年創立」是定位資訊（一句帶過），「林家鈺負債兩千萬在天母開四坪水餃攤」是品牌故事（不放分析引言）。

---

## 反模式：用品牌故事開場

分析文章用創辦故事開場有兩個問題：

第一，讀者要讀完整段故事才能判斷這篇跟自己的需求有沒有關係。分析型讀者的判斷依據是「分析什麼問題」而非「品牌起源有多勵志」——故事延遲了這個判斷。

第二，品牌故事暗示文章的 register 是「介紹一間公司」而非「拆解一個結構」。讀者期待後面會是公司簡介，而非毛利率差距的結構分析。開頭設定的 register 會框住讀者對整篇文章的預期。

## 判讀徵兆

| 訊號                                             | 該做的事                                                 |
| ------------------------------------------------ | -------------------------------------------------------- |
| 開頭第一句是年份 + 人名 + 創業動作               | 改成公司定位 + 規模 + 核心差異                           |
| 讀完第一段仍不知道「這篇要分析什麼問題」         | 把分析問題拉到第一段                                     |
| 開頭有情感修辭（負債翻身 / 從無到有 / 傳奇）     | 刪除，分析文章不需要情感投入                             |
| 同系列其他篇開頭用故事、某篇用分析定位、風格不一 | 以分析定位為準，不是以故事為準 — 分析型是系列的 register |
