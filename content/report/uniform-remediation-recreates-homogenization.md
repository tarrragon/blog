## 結論

為打破一個 cadence 模具而立的生成端規則，如果是「單一、均勻套用」的形態，會收斂出一個新模具、而且常比原模具更密。修掉「A 只說『…』；B 多了『…』」的分號模具時改成「段首一律目標詞先行」，套到整批稿件，就長出「目標詞 + 的X在/是『引號』」這個新模具——覆蓋率比原模具更高。

關鍵在自我感覺的遮蔽。套一條以「破模具 / 多樣化」為名的規則，在生成當下感覺像在製造變化，但「一律目標詞先行」本身就是一個單一模板。同源自審看不到，因為「我剛套了破模具規則」這件事替結果背了書。

---

## 為什麼均勻修法會複製出新模具

三層機制疊在一起：

1. **換模不等於破模**：把模具 A 換成規則 B、若 B 只有一種實現形態，產出的就是模具 B。cadence 同質化的成因是「同位置逐字同句型」、不是「用了哪個特定句型」——換一個統一句型只是搬家。
2. **「多樣化規則」偽裝成多樣化**：規則的意圖（破模具）跟規則的形態（單一模板）是兩回事。「目標詞先行」在意圖層是反同質化、在形態層卻是「目標詞 + 抽象名詞 + 是/在」這個唯一骨架。作者盯著意圖、放過形態。
3. **同源自審被「已修」背書遮蔽**：剛執行完一個以「破模具」為名的動作，自審會預設結果更多樣。這是 [#165](../register-violation-needs-cross-style-eyes/) 同源盲區在「修法後」的特例、比原始盲區更難察覺，因為多了一層「我明明修過」的信心。

## 這次的實證

一批擴充卡的 cadence，先前只跑過字源 fact-check、沒跑跨卡 cadence 異源 review。生成時我自套「目標詞先行」來避開先前修過的分號模具。整組 cadence reviewer 抓到三重復發：

| 生成端規則（意圖）                       | 均勻套用後的新模具（形態）                      | 密度                                               |
| ---------------------------------------- | ----------------------------------------------- | -------------------------------------------------- |
| 段首目標詞先行、避開「A 只說…；B 多了…」 | 「目標詞 + 的X在/是『引號』」                   | 12/16、warrant↔rigorous、viable↔resilient 逐字撞句 |
| 互指要標明反義 / 近鄰                    | 「跟 X 的差別在 Y：」+「它是 X 的反面——」雙骨架 | 近乎每張同位置掛一句                               |
| 字源收尾點出字面義                       | 「字面是『literal』——白話重述」破折號           | 約半數同位置                                       |

原始批雖有分號模具、但段首路數多（對照詞先行、目標詞引號、目標詞是）稀釋了節奏；均勻修法把少數幾例的手法複製成多數例、且沒有其他 framing 換氣——所以修法後比修法前更同質。只有整組跨卡異源 cadence reviewer 抓得到；生成當下的我、逐張看都覺得「有破模具」。

## 理想做法

### 修 cadence 用「輪替多個 framing」、不用「換一個統一模板」

準備多個替代 framing（動詞先行 / 情境先行 / 對照詞先行 / 目標詞引號），依序輪替分配。目標是位置層的多樣、不是找到一個更好的單一句型。判準：修完後段首起手法有幾種、而不是「有沒有避開舊句型」。

### 生成端規則本身要當成 cadence 候選、進整組異源 review

把新立的生成規則列進「這也可能同質化」的清單，修法後對整組跑一次 cadence frame（跨卡、同位置、逐字比對）。同源自審在「已修好」的錯覺下看不到、必須跨 item 整組看——cadence 是跨卡屬性、逐張視角結構上看不見。

### 修法後的 cadence 重掃是固定工序

「這批修完了」不是停止訊號。修法產物要重新過一次 cadence frame，這是 [#167](../remediation-introduces-sibling-variants/)「review scope 涵蓋修法產物」在 cadence 維度的應用。

## 沒這樣做的麻煩

### 修法後回報「破了模具」、新模具 silent

修完直接回報「破了模具、更多樣了」、不對整組重掃——新模具帶著「已優化」標籤留下。逐張自掃 keyword 會全 clean，因為同質化不在單卡、在整組連讀。

### 新模具往往比原模具更密

均勻修法把少數幾例的手法複製成多數例，連讀單調感反而比修前更強。修法製造了一個覆蓋率更高的模具。

### 靠「再想一個更好的統一句型」永遠在搬家

把希望放在「找一個更好的單一 framing」上、就一直在模具間搬家、不在破模。回到「位置層多樣」（成因層）才是解。

## 跟其他抽象層原則的關係

- **[#123 多重硬規範同時生效會把 cadence 推向便利解](../compliance-optimum-converges-cadence/)**：#123 是 emergent 版——N 個硬規範逼出一個「都通過」的 framing、批量複製到所有檔案；本卡是 deliberate 版——刻意立的「破模具規則」也會變成那個被複製的便利 framing。兩者同一個「單一 framing 佔滿整批」結局，成因一個是無意識收斂、一個是有意識修法反噬。
- **[#167 修法是新違規的來源、且常引入同類變體](../remediation-introduces-sibling-variants/)**：#167 是「修法漏放同一違規的下一個變體」（成因未動、變體浮現）；本卡是「修法本身裝上一個新模具」（不是漏、是新建）。都屬「修法產生新問題」，共享「review scope 要含修法產物」的推論，但 #167 是同違規殘留、本卡是新模具安裝。
- **[#122 Cadence 同質化是模板的隱形維度](../cadence-homogenization-in-batch-writing/)**：本卡是 #122 的動態版——#122 講 cadence 是模板的隱形維度（靜態辨識），本卡講「修 cadence 的動作會重建 cadence」（修法動態）。自檢一樣 grep 首句 / 段末 / 前導句，但對象要加上「修法後的產物」。
- **[#147 規範化跟自審是兩種認知任務](../rule-codification-vs-self-audit/)**：#147 講「立規範不保護同批稿件」；本卡更進一步——立的「生成規則」不但不自審、還主動把新模具寫進同批稿件。規範化的產物本身要進 audit。
- **[#165 register 違規：偵測可機械化、判定要靠文體異源的眼睛](../register-violation-needs-cross-style-eyes/)**：#165 講同源自審對 register 有結構上限、要異源；本卡把它推到「修法後」——修完的信心讓同源盲區更深，cadence 這種跨卡屬性尤其只有整組異源看得到。

## 判讀徵兆

| 徵兆                               | 該做的行動                                          |
| ---------------------------------- | --------------------------------------------------- |
| 為破某模具立了「一律 X」的生成規則 | 預期 X 本身變新模具、修完對整組重掃 cadence         |
| 修法後感覺「有破模具、更多樣了」   | 這感覺是同源盲區背書、逐張看不算數、要跨卡逐字比對  |
| 修法只準備了一個替代句型           | 換模不是破模、準備多個 framing 輪替                 |
| cadence 只在單卡自審、沒跑整組異源 | cadence 是跨卡屬性、同源逐張看不到、要整組 reviewer |

## 適用範圍與邊界

- **適用**：批量 / 系列寫作修 cadence 同質化、任何「立一條生成規則破某模式」的動作、multi-round-review 的 cadence frame 修法階段。
- **不適用**：修單卡孤立句型（無跨卡複製問題）；內容欄位層的規範（規模對照 / tripwire 這類有實質差異、不是純句型骨架）。
- **邊界**：「均勻修法會復發」不等於「不要立生成規則」——要立，但形態要輪替，且修法產物要進整組異源 cadence review。

## Self-case：本卡的觸發來源

本卡抽自 til/vocab 學術英文評論字擴充的 retrospective。原始 10 張跑過完整三輪 multi-round-review、修過「A 只說『…』；B 多了『…』」分號模具；後續兩批共 16 張擴充卡，我在生成端自套「段首目標詞先行」來避開那個模具、只跑字源 fact-check 就交付。補跑整組 cadence 異源 reviewer 時才發現：「目標詞先行」均勻套成「目標詞 + 的X在/是『引號』」新模具、覆蓋 12/16 張、比原模具更密，連互指句與字源收尾也各自骨化。修法（破 5 張開場 + 輪替 framing）後單卡 keyword 自掃全 clean，但那是逐張視角——同質化只在整組連讀浮現。教訓：修同質化的手法若是單一均勻模板、本身就會同質化，且同源自審在「已修」錯覺下看不到，要靠整組跨卡異源 review + framing 輪替。
