## 論述基礎與限制

本卡的論述基於 **1 個 batch**（電子業採購 planning 模組：6 篇文章，每篇 43-50 行、4-6 個 H2、每個 H2 約兩段）抽出來的觀察。使用者對該模組的判定包含「每篇文章的描述節奏跟內容應該要重新評估拆分」。具體限制：

- **分析弧五步是責任清單、不是模板**：情境 → 機制 → 量化取捨 → 判準 → 失效條件的順序可調、步驟可合併——把它硬套成固定章節結構會複製 cadence 同質化。
- **深度基準是相對的**：本 batch 用同分類 sibling（案例拆解文章）當深度錨點；分類沒有 sibling 時，錨點要另找（讀者的下游任務需要多深）。
- **單 batch 觀察**：淺層節奏的具體症狀（H2 數、段落數）是這批的形態，其他 batch 可能以不同表面形態呈現同一結構問題。

## 核心原則

**教學模組的文章邊界由「一個分析責任走完完整分析弧」決定——情境、結構與機制、量化取捨、判準、失效條件。** 判斷一個主題該獨立成篇還是併入他篇，看素材夠不夠支撐走完這條弧：夠，成篇；不夠，併入相鄰主題當一段、或標 backlog 等素材補齊。這個判準讓每篇文章都有縱深——讀者離開時帶走的是一個主題的判斷力，而不是六個主題的印象。

把來源素材的主題聚類直接當文章邊界，是反向的 sizing：先數 source 有幾個主題群、一群一篇，再把每群的句子攤進去。產出的形態是全模組均勻的淺層節奏——每篇 4-6 個 H2、每個 H2 兩段，每個主題都開了場、沒有一個走完弧。深度被結構鎖死：40 行塞六個主題的骨架，之後想補深某個主題，只能重拆。

本卡跟原子化的 focus 完整度判準同構：focus 完整度問「這張卡把它聲稱的議題講完了嗎」，本卡在教學模組層給「講完」一個操作定義——分析弧五步有沒有走完。

## 具體 case

### case 1：均勻淺層節奏

採購模組 6 篇全部落在 43-50 行、4-6 個 H2、每 H2 兩段——模組級的均勻分佈本身就是「按主題攤平」的指紋。對照同分類的案例拆解文章：單一事件寫超過 100 行、含成本算式與市場結構分析。sizing 邏輯不同（前者按 source 主題數均分、後者按分析責任展開），深度差一個量級。

### case 2：素材句子跨篇漂移

同一句來源素材出現在多篇：「地震颱風隔天問廠商產能」在 planning 心態篇與職場生態篇各出現一次、「搶單旺季 forecast 測不準」在心態篇與備料篇各一次。按主題攤平時，一句素材跟多個主題都沾邊，就被複製到多篇；按分析責任拆分時，每個機制有唯一的歸屬篇章，素材跟著機制走、重複自然消失。

### case 3：genre 混裝

「不可能三角與採購職場生態」一篇裡，前半是可分析的取捨結構（交期 / 價格 / 品質的談判空間、通路鎖定下的議價對象轉移），後半是職場心態（平常心、身體健康、面對外界評價）。兩種知識類型（分析教學 vs career essay）互相稀釋——分析讀者讀到後半覺得內容失焦，心態內容也因為擠在分析文章裡只能各給兩段。這是拆分階段的路由問題：兩種知識類型該在成篇前就分開。

## 沒這樣做的麻煩

### 讀者每個主題都被開場、沒有一個學到判斷力

淺層節奏的閱讀體驗是連續的「引入——初步展開——切下一個」。讀者記得模組提過哪些概念，但任何一個概念要在自己的情境使用時，發現文章停在概念介紹、判準與失效條件都沒展開。

### 重複素材的維護成本

同句素材散在多篇，之後修正（發現表述不精確、來源更新）要多處同步，漏一處就出現版本分歧。

### 補深必須重拆

發現某主題需要加深時，40 行 6 主題的骨架沒有空間——插入完整的機制推導會讓該篇比例失衡，正確做法是把該主題抽出成篇，等於當初的拆分作廢重來。拆分階段多想一步「這個主題值不值得一條完整的弧」，省掉整輪重構。

## 跟其他抽象層原則的關係

- **[#211 複合問題先拆機制再談交互](../compound-problem-decompose-then-interact/)**：互補的兩個方向。#211 管「跨篇怎麼拆」（每個機制獨立成篇、交互用連結串接）；本卡管「每篇拆出來之後要多深」（一篇走完一條分析弧）。合起來構成教學模組的結構決策：先按機制拆邊界、再按分析弧驗深度。
- **[#209 教鑑別能力、不教操作流程](../teach-judgment-not-procedure/)**：分析弧是判斷力導向在篇章結構上的展開——機制當主線（弧的中段）、判準與失效條件是判斷力的交付物（弧的尾段）。按主題攤平的淺層節奏結構上到不了判斷力：機制段只有兩段的空間。
- **[#122 cadence 同質化是模板的隱形維度](../cadence-homogenization-in-batch-writing/)**：均勻淺層節奏是 cadence 同質化的模組級形態——同骨的不只是句式，還有每篇的「H2 數 × 段落數」骨架。#122 的偵測方法（整組連讀、單篇 review 看不到）同樣適用：逐篇看每篇都「正常」，攤開全模組的行數與 H2 分佈才看見均勻得可疑。
- **[#212 SRP 違反是路由訊號、不是刪除訊號](../misplaced-content-needs-route-not-deletion/)**：case 3 的 genre 混裝適用 #212 的處理——心態內容不是該刪，是該路由到自己的篇章並標明知識類型。本卡把這個路由判斷提前到拆分階段：成篇前就依知識類型分流，省掉成篇後的搬遷。

## 判讀徵兆

| 徵兆                                                | 該做的行動                                                     |
| --------------------------------------------------- | -------------------------------------------------------------- |
| 全模組行數與 H2 數均勻（每篇 40-50 行、4-6 H2）     | 按主題攤平的指紋——逐篇問「哪個 H2 值得一條完整的弧」、重估拆分 |
| 同一句來源素材出現在 2+ 篇                          | 素材跟著主題漂移——找出它支撐的機制、收斂到機制的歸屬篇章       |
| 一篇內混兩種知識類型（分析 vs 心態 / 教學 vs 記錄） | genre 混裝——依知識類型分流成篇、各自標明定位                   |
| 對照同分類 sibling、篇均深度差一個量級              | house style 失配——用 sibling 當深度錨點重估每篇的分析弧        |
| 想補深某主題時發現骨架沒空間                        | 拆分已被淺結構鎖死——把該主題抽出成篇、重跑拆分而不是塞段落     |

## 適用範圍與邊界

- **適用**：教學模組的文章邊界規劃（成篇前）、或模組被判定「節奏淺、每篇都蜻蜓點水」時的重構診斷（成篇後）。
- **邊界**：
  - **知識卡片豁免**：卡片的責任是單一術語的可查閱定義，本來就短，分析弧不適用；卡片的深度要求另有建卡判準。
  - **hub / 索引頁豁免**：`_index` 是路由頁，責任是分流不是縱深。
  - **弧是責任清單不是模板**：五步可以調序、合併，操作型文章的「量化取捨」可以體現為成本對照而非算式；但承擔判準責任的文章，「機制」與「失效條件」兩步無可豁免。
  - **短篇不等於違規**：一個機制簡單、素材完整的主題可以 60 行走完弧——判準是弧完整、不是行數達標。

## Self-case：本卡的觸發來源

觸發於採購 planning 模組的拆分檢討。模組 6 篇由來源經驗談的主題聚類直接切出（心態 / 供應商 / 備料 / 追料 / 三角與生態 / 原廠規則），每篇 43-50 行、4-6 H2、每 H2 兩段，全模組均勻。使用者判定「每篇文章的描述節奏跟內容應該要重新評估拆分」。

回查證據：三組素材句跨篇重複（grep 實測）；「不可能三角」篇前半分析後半心態的 genre 混裝；對照同分類案例拆解文章（單一事件 100+ 行含算式）深度差一個量級。素材端根因（機制層未重建）見 [#216](../anecdotal-source-needs-mechanism-reconstruction/)、審查端缺口（三輪未抓到）見 [#217](../review-needs-claim-support-frame/)——本卡記結構端：就算機制補齊，按主題攤平的骨架也裝不下，拆分判準要先換成分析弧。

**Update（2026-07-10）**：本卡觸發的採購模組已依分析弧重新拆分——6 篇攤平結構改為 8 篇各承擔一個分析責任（放量分配獨立成篇、職場心態分流成標明知識類型的獨立篇）、行數與深度隨責任變動、重複素材句收斂完成。卡內的均勻節奏描述以重寫前版本為準。
