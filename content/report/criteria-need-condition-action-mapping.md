## 論述基礎與限制

本卡的論述基於採購模組重寫過程中「同一個坑掉了兩次」的觀察：重寫前的版本判準停在口訣層（被 #216 診斷）；重寫後機制全部重建、其中一段判準仍停在維度清單層，被 Round 3 的 self-application reviewer 用分析弧檢查抓到——證明這是機制重建之後仍會殘留的獨立缺陷。具體限制：

- **三層分級是這個 batch 的觀察歸納**，可能有中間形態（例如帶權重的維度清單）；分級的用途是判讀不是分類學。
- **重算測試的「參數」依內容型態而異**：分析文是數字與情境參數、操作文是環境與工具、決策文是組織約束——測試形式要跟著換。
- **單 batch 觀察**。

## 核心原則

**教學文章的判準要寫到條件層：條件到行動的映射，加上映射失效的情境。** 判準有三個成熟度，只有第三層交付判斷力：

| 成熟度   | 形態                                     | 讀者拿到什麼                   |
| -------- | ---------------------------------------- | ------------------------------ |
| 口訣     | 無推導的結論（「單一家不超過七成」）     | 一條要背的規則、換情境即失效   |
| 維度清單 | 「判斷看 A、B、C」                       | 知道要考慮什麼、推不出該做什麼 |
| 條件映射 | 「A 成立 → 做 X；A 破 → 切 Y」＋失效情境 | 帶自己的參數能得出行動         |

維度清單最有欺騙性：它長得像判準（有「判斷」「取決於」這類判準動詞）、通過字句層與機制層的審查（每個維度都有機制支撐）、卻在讀者真正要做決定的那一刻斷線——「我知道要看毛利結構、調價機制、時間差，所以我該吸收還是轉嫁？」答不出來。它是判準的空殼。

驗收用**重算測試**：讀者帶著自己的參數走進這段判準，能不能走出一個行動。走得出來，判準到位；走出來的是「我需要再想想」，這段還停在第二層。

條件無法窮舉的決策有一個合法的第三層變體：**自查問句組**——把決策變數轉成讀者能自己回答的問題（「過去幾季的 forecast 兌現率多高、有沒有綁定架構、佔對方營收比重多少」），問完答案自然排出行動優先序。它跟維度清單的差別在可答性：維度清單給名詞（「訂單信用」）、自查問句給讀者答得出來的問題。

## 具體 case

### case 1：同一段判準的三個版本

**口訣層**（重寫前）：「量產初期放量不會讓單一家超過約七成。」——數字無推導，換情境失效。

**條件映射層**（重寫後的放量分配篇）：「先問主力失效時能容忍的斷供時間是幾天（由庫存決定），再問這段時間內替補能爬到多少（由產能型態決定），兩個答案夾出主力的上限」＋參數表（替補型態 / 庫存厚度 / 家數各自怎麼改變上限）＋邊界（獨家料不適用、小量期單源量產雙源認證）。讀者換參數可重算。

### case 2：機制重建後仍停在第二層的段落

重寫後的漲價篇「吸收還是轉嫁」段，初版寫「判斷的維度是毛利結構、下游調價機制、時間差」——三個維度各有機制支撐、字句合規、通過了機制與一致性審查，Round 3 self-application reviewer 才指出「判準步只有形式沒有實質」。修正後補上映射：「下游合約帶成本連動條款的，轉嫁優先；毛利吃得下且客戶議價力強的，吸收換長期關係；兩頭都緊的走部分轉嫁」＋時間參數（轉嫁窗口由下游調價頻率決定、年度議價客戶 squeeze 可拖四季）。這個 case 的價值在時序：**機制重建完成不保證判準到條件層**，兩者是獨立的檢查。

### case 3：自查問句組作為第三層變體

配給順位的判準無法寫成通用條件式（每家供應商的權重不同），改交付四個自查問句（forecast 兌現率 / 綁定架構 / 營收占比 / 窗口熟悉度）加一條行動規則（「哪一項答得最虛、哪一項就是最該補的投資」）——條件不可窮舉時，可答的問題加排序規則同樣走得出行動。

## 沒這樣做的麻煩

### 讀者在決策點斷線

教材的存在理由是讀者將來要做那個決定。維度清單把讀者送到決策點門口然後放手——最需要幫助的一步剛好缺席。

### 審查給假陰性

維度清單通過大多數審查 frame（字句合規、機制有支撐、結構有判準段），只有拿「重算測試」去壓它才現形。審查清單裡沒有這一條，空殼判準就一路活到讀者手上。

### 空殼會被複製

批次寫作時，一段「判斷看 A、B、C」的句式合格又好寫，容易被當成判準段的模板複製到 sibling——空殼以 cadence 的形式擴散。

## 跟其他抽象層原則的關係

- **[#218 文章按分析弧拆分](../article-split-by-analysis-arc-not-source-spread/)**：本卡是 #218 分析弧「判準步」的驗收標準。弧檢查問「有沒有判準步」，本卡問「判準步是不是實質的」——case 2 正是弧存在、步空殼的實例。
- **[#210 壓縮結論剝奪推導路徑](../compressed-conclusion-strips-derivation/)**：三個成熟度可以讀成壓縮程度：口訣是壓掉全部推導、維度清單是壓掉「條件怎麼組合成行動」的後半段推導。#210 的修法（還原推導）在判準段的具體形式就是補條件映射。
- **[#209 教鑑別能力、不教操作流程](../teach-judgment-not-procedure/)**：#209 定方向（教判斷力），本卡定顆粒度（判斷力交付到哪一層才算數）。條件映射不是把判斷力退化成流程——映射帶著機制與失效情境，讀者知道條件為什麼指向那個行動。
- **[#216 經驗談來源要重建分析層](../anecdotal-source-needs-mechanism-reconstruction/)**：#216 修口訣層（重建機制）、本卡修維度清單層（機制到行動的最後一哩）——case 2 證明兩者獨立：#216 做完、本卡的缺陷仍可能存在。

## 判讀徵兆

| 徵兆                                                       | 該做的行動                           |
| ---------------------------------------------------------- | ------------------------------------ |
| 判準句的動詞停在「看 / 考慮 / 取決於」、沒有「→ 就」的映射 | 維度清單訊號——補條件到行動的映射     |
| 讀者問「所以我該怎麼做」、這段答不出來                     | 重算測試失敗——判準還在第二層         |
| 判準段沒有失效情境                                         | 映射缺邊界——補「條件破了切到哪」     |
| 條件寫不成通用式                                           | 換自查問句組＋排序規則、不硬湊假條件 |
| 批次稿件的判準段句式雷同（「判斷看 A、B、C」連發）         | 空殼正在被當模板複製——逐段跑重算測試 |

## 適用範圍與邊界

- **適用**：承擔判準責任的教學段落——分析文章的決策段、選型指南、runbook 的分支點。
- **邊界**：
  - **背景與機制段不適用**：不是每段都要交付行動；本卡管的是「文章承諾給判準」的那幾段。
  - **維度清單有合法用途**：作為條件映射的前導（先列維度、再給映射）合規；違規的是列完維度就收尾。
  - **探索型內容豁免**：record / 思考筆記型內容記錄的是未收斂的思考，不強制到第三層——但要標明知識類型，別穿教學的衣服。

## Self-case：本卡的觸發來源

觸發於採購模組重寫的 retrospective：同一個缺陷在兩個版本各出現一次。重寫前，「不超過七成」式的口訣層判準被 #216 診斷、重寫時全數還原成機制；重寫後，漲價篇「吸收還是轉嫁」段仍停在維度清單層，通過了機制正確性與一致性審查、Round 3 self-application reviewer 以「判準步只有形式沒有實質」抓出。兩次的共同點是「看起來像判準的東西通過了審查」，差別是形態升了一級——說明判準成熟度是獨立於機制重建的檢查軸，值得單獨立卡、進審查清單。
