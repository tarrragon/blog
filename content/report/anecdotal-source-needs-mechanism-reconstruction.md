## 論述基礎與限制

本卡的論述基於 **1 個 batch**（電子業採購 planning 模組：6 篇文章 + 13 張知識卡，來源為採購從業者的社群經驗談）抽出來的觀察。該模組經三輪 multi-round review 修訂後 merge，使用者仍判定「寫法粗糙、是講故事不是商業分析教學」。具體限制：

- **機制重建的知識來源有兩條**：領域通用理論（供應鏈經濟學、選擇權思維、通路結構）或回頭追問原始從業者。本卡不裁定哪條優先——看素材形態與作者對該領域理論的掌握度。
- **重建深度的停止線未量化**：判準是「承擔判準責任的斷言」才需要重建機制，氛圍性敘述刪除或降為 hook 即可；但「承擔判準責任」的邊界仍靠逐句判讀。
- **單 batch 觀察**：症狀清單（訴諸權威、無推導閾值、anecdote 跨篇重複）來自這一批，後續 batch 可能補充新形態。

## 核心原則

**教學轉換處理經驗談 source 時，產出的知識類型要從「敘事」升級成「分析」：每個承擔判準的斷言，都要能回答「為什麼成立、在什麼條件下不成立」。** 經驗談是從業者走完推導後輸出的最後一步——「量產初期單一家放量不超過七成」「備量要覆蓋一整個交期」「絕不斷料」都是有效經驗，但推導路徑留在從業者腦中。轉換的核心工作是重建這條路徑：用成本結構（停線成本對比呆料持有成本）、誘因結構（代理商為什麼接受 ship and debit）、市場結構（缺料市場的搶單賽局）把斷言變成可推導、可質疑、可遷移的機制。

只做系統化整理——分主題、換書面語、加表格、補術語卡——會產出「披著教學結構的經驗談」：結構合規、字句合規、結論都在、支撐全缺。這種產物通得過字句層與結構層審查，但讀者拿到的是「要相信的規則」，不是「能自己推的判斷」。

轉換的分層操作（source layering 的經驗談版）：

| source 內容層                                           | 教學轉換的處理方式                          |
| ------------------------------------------------------- | ------------------------------------------- |
| 事實（規則、流程、事件：NCNR 條款、NPI 流程、EOL 通知） | 保留，構成教材的 fact 層                    |
| 從業者判讀（「不超過七成」「交期決定備量」）            | 當 hypothesis，重建支撐機制後才升為教材判準 |
| 分析層（機制、量化、結構解釋）                          | source 沒有自帶，由作者從領域通用知識補建   |

跟「外部分析文章」source 的關鍵差異：分析師文章自帶分析層（frame、機制、數據），轉換工作是**拆層**（分開事實、原作者判讀、本文推導）；經驗談只有事實與判讀兩層，分析層要**重建**。判別 source 類型，決定轉換流程走哪一條。

## 具體 case

### case 1：代價不對稱被斷言、沒被分析

**經驗談版**：「斷料一旦發生，代價遠高於多備一點的呆料成本……正因為代價不對稱，planning 的每個動作都在斷料這條線之前留餘裕。」

**問題**：「代價不對稱」是整個模組的核心判準，但它只被宣告。停線的成本形狀（產線人力與設備閒置、交貨違約金、客戶轉單的終身價值損失）對比呆料的成本形狀（現金佔用的資金成本、跌價、報廢）——量級差距的分析才是「留餘裕」的推導基礎。缺了這層，讀者無法回答下一個問題：「餘裕該留多少才不算過度？」

**分析版的走向**：把兩側成本各自展開成可估算的組成，給出量級對比（哪怕是示意性的算式），「不對稱」從口號變成讀者可以帶著走的估算框架。

### case 2：magic number 無推導

**經驗談版**：「量產初期放量時，不會讓單一家就超過七成。」

**問題**：七成是從業者的壓縮結論。它實際上是「單一家量產爬坡失敗時，其餘供應商短期能吸收的產能上限」的反推值——比例跟著替補家的產能彈性變動。講機制之後，數字降級為例示，讀者在自己的情境（替補家更弱或更強）能重算出自己的比例。

### case 3：操作背後的分析語言缺席

**經驗談版**：「只要在截止日前可以無條件修改或取消，就把常用料號直接下單卡到一兩年後的遠期。」

**問題**：這個操作的分析本質是**免費選擇權**——下單取得未來取貨的權利，取消成本趨近零，到期前可放棄。用選擇權語言重建之後，「為什麼 NCNR 的膽量要收」（權利金變成全額買斷）、「為什麼原廠要設改單截止日」（發行端控制未定價風險敞口）都能從同一個機制推出來。source 給了操作，分析語言由作者補建。

### case 4：訴諸權威當論據（反面對照）

「厲害的採購對料件的掌握度不輸研發」在同一模組的三篇文章裡當論據使用。訴諸從業者權威是經驗談自己的支撐方式——教材沿用它，等於知識類型沒有升級：讀者被要求相信「厲害的人都這樣」，而不是理解「為什麼懂料改變議價與預判的結構」。

## 沒這樣做的麻煩

### 讀者拿到規則、帶不走判斷

規則綁著來源情境的隱含參數（產業、規模、供應商結構）。情境一變，規則失效，讀者沒有機制可以重算。這正是教學價值的核心損失——教材的存在理由是給判斷力，規則清單在原始經驗談裡就有了。

### 教材跟分類定位失配

放在商業分析分類下的模組，用不上該分類自己的分析語言（成本結構、單位經濟、通路 margin、市場賽局）。同分類的 sibling（案例拆解文章有算式、有市場結構分析）跟新模組的知識類型斷層，讀者從 sibling 讀過來會立刻感到深度落差。

### 審查抓不到

字句層、cadence、讀者旅程審查都在文字表面操作，知識類型錯位全數通過——每一句都合規、整體卻是經驗談。這個審查盲區獨立成卡（見 [#217 審查要有斷言支撐 frame](../review-needs-claim-support-frame/)）。

### 來源句子被攤平複用

沒被消化成機制的素材只能被重複引用。本 batch 的實測：「地震颱風隔天問廠商產能」出現在兩篇、「搶單旺季 forecast 測不準」出現在兩篇、「三不五時看市場新聞」出現在兩篇。素材消化成機制後，每個機制有自己的歸屬篇章，重複自然消失。

## 跟其他抽象層原則的關係

- **[#143 外部分析文章要先拆成事實、作者判讀、本文推導](../external-analysis-source-layering/)**：sibling、按 source 類型分工。#143 處理「自帶分析層的 source」（分析師文章有 frame，工作是拆層防止原作者判讀被當事實）；本卡處理「沒有分析層的 source」（經驗談只有事實與判讀，分析層要重建）。轉換流程的第一步是判別 source 類型，決定用哪張卡。
- **[#210 壓縮結論剝奪推導路徑](../compressed-conclusion-strips-derivation/)**：經驗談斷言正是「別人的壓縮結論」。#210 講作者自己走完推導只輸出最後一步；本卡講引用**他人**已壓縮的結論——修法同構（還原推導路徑），但推導的來源不同：#210 從作者自己腦中還原，本卡要從領域理論補建或回頭追問從業者。
- **[#209 教鑑別能力、不教操作流程](../teach-judgment-not-procedure/)**：判斷力導向的文章把機制理解當主線。經驗談 source 不重建機制，就只有規則可寫——結構上做不到判斷力導向。本卡是 #209 在「素材端」的前置條件：機制不存在時，先重建才有主線可用。
- **[#118 Standard-driven vs Case-driven 領域判讀](../standard-driven-vs-case-driven-domain-judgment/)**：經驗談是 case 素材的一種形態。#118 判斷「該用 case 還是 standard 驅動」，本卡補「case 素材進教材前的知識類型升級」——case-driven 不等於把 case 的敘事口吻搬進教材。

## 判讀徵兆

| 徵兆                                                      | 該做的行動                                           |
| --------------------------------------------------------- | ---------------------------------------------------- |
| 教材斷言的支撐是「厲害的 / 資深的人都這樣做」             | 訴諸權威即知識類型未升級——重建該斷言背後的機制       |
| 數字閾值（七成、兩家、覆蓋一個交期）無推導                | 找出閾值背後的反推邏輯，把數字降級為機制的例示       |
| 同一句 anecdote 出現在 2+ 篇                              | 素材未消化——找出它支撐的機制、收斂到該機制的歸屬篇章 |
| 規則被描述、但「規則為什麼存在」缺席（如 ship and debit） | 補設計者的誘因分析：誰設的規則、保護誰的什麼利益     |
| 模組所在分類有成熟分析語言、模組完全用不上                | genre 失配訊號——回頭檢查轉換是否只做了主題化與書面化 |

## 適用範圍與邊界

- **適用**：從業者經驗談、訪談稿、口述整理、社群貼文轉教學模組——特別是目的地是分析導向分類（商業分析、架構決策、選型教材）時。
- **邊界**：
  - **心態與職場生態內容**（平常心、關係經營的個人修為）是合法的經驗分享，但知識類型是 career essay——分開成篇並標明定位，或壓縮成分析文章的一段背景，處理方式是路由不是重建。
  - **操作型 runbook**（追料節點的介入選項清單）可以保留操作形式，但選項的排序邏輯（為什麼從代價低的開始）仍要有機制支撐。
  - **重建有停止線**：只有承擔判準責任的斷言需要機制；把每句經驗談都經濟學化會把教材寫成論文，氛圍性敘述刪除或降為 hook。

## Self-case：本卡的觸發來源

觸發於電子業採購 planning 模組（6 篇文章 + 13 張卡，來源為採購從業者社群經驗談）。模組經三輪 multi-round review（compliance / cadence + 讀者旅程 / steelman + outbound）修訂後 merge，使用者判定「寫法太粗糙——不是要用講故事的方法做教學，需要的是商業分析角度的教學文章」。

回查證據：「厲害的採購不輸 RD」在三篇當論據；「單一家不超過約七成」「代價不對稱」「交期決定備量」等核心判準全數無推導；ship and debit 只描述規則、未分析原廠與代理商的通路誘因；「可無條件取消就下遠期單」給了操作、缺選擇權的分析語言；同批 anecdote 句子跨篇重複（grep 實測三組 ×2）。轉換只做了主題化、書面化、術語卡化——分析層從未被建出。
