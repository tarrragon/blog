## 論述基礎與限制

本卡的論述基於 **1 個 case**：VM/Hyprland 系列的寫作中，我反覆用「坑」這個字（「實測踩到的坑」「命中坑」「這個坑」），台灣讀者點出兩件事——繁中 IT 圈少用「坑」，且同一個泛用詞反覆出現讓文章變扁平。具體限制：

- **「坑」是「泛用濫用」+「地區偏移」的雙重問題**：本卡聚焦前者（泛用詞被反覆濫用、跨地區都成立的寫作問題），地區偏移面（「踩坑 / 避坑」是簡中技術社群高頻詞）歸 [#112 地區用語](../regional-terminology-alignment/)。
- **替代詞清單是偏好、不是硬規則**：某些情境「坑」仍最貼切（引號內引用、口語 hook）。清單是「換個更精確的詞」的起點、不是查表替換。
- **「詞彙變化提升記憶」無量化資料**：本卡描述「具體詞讓讀者記得更清楚」是直覺論述加讀者回饋、未做使用者測試。

## 核心原則

一個泛用詞反覆使用，會把本來不同的情境壓成同一個模糊標籤。每個情境有它自己精確的詞——把「坑」拆開來看：非預期的行為是**意外**、埋著等你踩的前提是**陷阱**、當下的故障是**出問題 / 故障**、狀態異常是**發生狀況**。用精確的詞，讀者對每個情境的印象清楚且各自獨立；用同一個泛用詞蓋過去，這些情境在讀者腦中糊成一片。詞彙的變化本身就是資訊——它告訴讀者「這幾件事不一樣」。

## 為什麼泛用詞會被濫用

- **省思考**：泛用詞什麼都能套，用它就不必停下來想「這個情境到底是哪一種」。精確詞逼你先分類。
- **LLM 自產傾向**：模型高頻詞（坑 / 東西 / 搞 / 處理）填空成本低、順手就出來。
- **網路語感滲透**：「踩坑 / 避坑」在簡中技術社群是高頻詞、繁中原本少用，讀多了會不自覺帶進來。
- **用順了就整篇重複**：這是 [#122 cadence 同質化](../cadence-homogenization-in-batch-writing/) 在字詞層的體現——一個詞用順了，整篇都是它。

## 「坑」的替代詞（依情境選）

| 情境                    | 別用「坑」、用   |
| ----------------------- | ---------------- |
| 非預期的行為 / 結果     | 意外、非預期行為 |
| 埋著等人踩的設計 / 前提 | 陷阱、地雷       |
| 當下的故障              | 出問題、故障     |
| 狀態異常                | 發生狀況、異常   |
| 難處理的細節            | 眉角、難處、細節 |

判準不是背這張表，是先問「這裡具體是什麼」——答案出來，精確的詞自然浮現。

## 更廣：被濫用的泛用詞不只「坑」

同一類的還有「東西」「搞」「弄」「處理」「做」「情況」。共同特徵是：一個泛用詞在一段或一篇裡反覆出現、而每次指的其實是不同的具體事。判準不是「這個詞壞」，是「同一個泛用詞被拿來蓋過本該分開的具體情境」。看到密集重複就回頭問：這裡具體是什麼，換成那個具體詞。

## 何時泛用詞仍然合理

- 真的就是泛指、沒有更精確的詞可換時。
- 引號內引用原話（別人說了「坑」，照引不改）。
- 口語 hook / narrative 的輕度使用、不主導段落時。

## 跟其他抽象層原則的關係

- [#112 地區用語對齊](../regional-terminology-alignment/)：「坑」有地區偏移面（簡中高頻、繁中少用），那一面歸 #112；本卡聚焦跨地區都成立的「泛用詞濫用」面。兩張卡在「坑」這個詞上交集、但管的軸不同（地區 vs 泛用）。
- [#111 口語修辭稀釋技術精度](../colloquial-rhetoric-erodes-technical-precision/)：泛用詞常伴隨口語感；#111 管「口語 vs 精確」、本卡管「泛用 vs 具體」，是同一個字句層的相鄰兩軸。
- [#122 cadence 同質化](../cadence-homogenization-in-batch-writing/)：同句型反覆是 cadence、同一個詞反覆是它在字詞層的版本；本卡是 #122 從句型下沉到用詞的具體化。
- [#149 命中是候選不是判決](../keyword-bank-hit-is-candidate-not-verdict/)：grep 到「坑」是候選、不是判決——引號內引用、真泛指合規，仍要一個語意判定步驟。

## 判讀徵兆

- 一篇文章「坑」出現 3 次以上、而且每次指的是不同的事。
- 讀完記不清「那些坑」各自是什麼——它們被壓成同一個模糊印象。
- 字句層 grep：`rg "坑|東西|搞|弄|處理一下|情況" <files>` 命中密集、且集中在同一批段落。

## Self-case：本卡的觸發來源

這張卡是讀者回饋直接觸發的：VM/Hyprland 系列我在多篇反覆寫「實測踩到的坑」「命中坑」「這個坑」，台灣讀者指出繁中不說「坑」、且反覆用同一個泛用詞讓文章扁平，建議依情境改用「意外 / 出問題 / 發生狀況 / 陷阱」等精確詞。回饋同時揭露了比「坑」更廣的原則——泛用詞濫用是跨詞、跨地區的字句層問題。
