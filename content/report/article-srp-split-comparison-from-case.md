## 論述基礎與限制

本卡的論述基於八方雲集案例分析的結構拆分。初版文章同時承擔兩個分析責任：八方自身的商業模式與財報判讀，以及八方與揚秦的跨公司結構比較（八維比較表 + 獲利結構比較表 + 散落在各段的比較句）。拆分後三個效果立即可觀察。限制：觀察來自單一次拆分，但原則適用於所有「個體分析 + 跨個體比較」並存的文章結構。

---

## 核心原則

**個體的深度分析和跨個體的結構比較是兩個獨立的分析責任——各自有不同的讀者需求、不同的擴充方向、不同的資料組織方式。** 把兩者塞在同一篇文章裡，兩邊都做不深：案例分析的篇幅被比較內容佔用，比較內容被鎖在單一公司的敘事脈絡裡無法獨立擴充。

拆分後的三個效果：

1. **案例文章深化**。八方文章移除比較段後，騰出空間深化營業槓桿的雙面性（正向放大 5.2 倍 + 反向風險）、德州工廠 300 產能 vs 13 間店的閒置壓力、同店來客數 vs 營收成長率的判讀差異、丹堤和芳珍蔬食的定位問題——這些分析在原版被比較表格擠掉了。
2. **比較文章可擴充**。比較篇獨立後，之後加入三商餐飲（7705）或其他加盟母公司只需要在既有比較軸加行，不需要改動任何一篇案例文章。
3. **被引用的文章也深化**。揚秦文章原本用兩段解釋八方的模式來說明毛利率差距；拆分後改為引用比較篇，騰出空間深化揚秦自身的關係人交易判讀方法（年報附註查超秦採購比例 → 比對超秦毛利率 → 判斷改善來源）和展店落差的三面向分析。

## 反模式：在案例文章裡內嵌完整比較

案例文章裡出現「跟 X 的比較」段落有兩種情境：

**比較服務案例的論點**——用一句「同業 X 的毛利率 36%，差距 9pp」建立參考基準，讓讀者理解本案例的相對位置。這種引用是案例分析的一部分，佔一兩句，不需要拆分。

**比較本身是獨立的分析**——用完整的多維比較表、逐維度拆解差距原因、得出結構性結論。這已經不是「引用基準」而是「做比較分析」，承擔的是跨公司比較的分析責任。這種段落嵌在案例文章裡會造成兩個問題：案例文章的讀者被迫讀完比較才能繼續、比較內容的讀者要先進入一家公司的脈絡才能找到比較表。

判別方式：刪掉比較段落後，案例文章的分析是否仍完整。如果完整，比較段應該獨立成篇。

## 判讀徵兆

| 訊號                                                              | 該做的事                                                          |
| ----------------------------------------------------------------- | ----------------------------------------------------------------- |
| 案例文章出現完整的跨公司比較表（3+ 維度）                         | 評估比較段是否能獨立成篇                                          |
| 同一段分析在兩篇案例文章中重複（A 文章解釋 B 的模式、B 也解釋 A） | 比較內容應該只在比較篇出現一次，兩篇案例各自引用                  |
| 想加入第三家公司的比較，但不知道放在哪篇案例文章裡                | 比較內容已經超出任何單一案例的責任——需要獨立的比較篇              |
| 刪掉比較段後案例文章的分析仍完整                                  | 比較段是附加的、不是案例的必要部分——拆出去                        |
| 案例文章的某段分析沒有足夠空間深化                                | 檢查是否有其他責任的段落佔了篇幅——可能是嵌入的比較、背景、或 WRAP |
