## 論述基礎與限制

本卡的論述基於採購 planning 模組重寫的 before / after 對比（1 個 batch）：重寫前 6 篇是主題集合（心態 / 供應商 / 備料 / 追料 / 三角 / 規則平行排開、互相引用但沒有共同基準），重寫後 10 篇是推導體系（成本不對稱當源頭、每篇承擔一條展開）。具體限制：

- **「源頭」的形態未窮舉**：本 batch 的源頭是一個成本結構（斷料對呆料的量級不對稱），其他模組的源頭可能是一個約束（CAP 定理之於分散式教材）、一個生命週期（請求的一生之於 web 教材）——共同點是「各篇判準能折算回去的基準」，形態不限於成本。
- **收益的觀察是間接的**：跨篇矛盾被抓到、擴篇順利掛載，都發生在有源頭的版本上，但沒有對照組證明「無源頭就抓不到、掛不上」——推論來自機制（矛盾要有共同基準才成立為矛盾），不是控制實驗。
- **單 batch 觀察**。

## 核心原則

**分析教學模組的結構要是推導體系：一個源頭機制，每篇承擔一條從源頭展開的推導線；模組索引能一句話說出「推導的起點是什麼」。** 主題集合式的模組——把相關主題平行排開、各講各的——每篇可以各自合格，模組卻只是目錄：判準之間沒有共同基準，深度想補也沒有「往哪裡深」的方向。

源頭機制買到四件事，每件都在 batch 裡實際兌現過：

**判準有共同的尺**。備多少（覆蓋交期）、養不養第二家（保費對期望損失）、急救划不划算（溢價對停線）、呆料何時停損（殘值對持有成本）——表面是四個主題的四套判準，全部折算到同一個成本不對稱上。讀者學到的是一把尺量到底，不是四套要分開記的規則。

**跨篇矛盾才會現形**。「可取消訂單下滿卡位」與「取消紀錄折損配給順位」分屬兩篇、各自機制正確，合起來互斥——這個矛盾之所以「是矛盾」，是因為兩篇的判準都掛在同一個體系上主張效力；主題集合裡同樣的兩句話只是兩篇各自的說法，審查在結構上就抓不到。

**新篇有掛載點**。模組後續擴充漲價應對與呆料處置兩篇時，掛載動作是明確的：漲價的籌碼力量折回第二來源的保費框架、呆料的停損折回不對稱的呆料側。主題集合的擴篇只能「再加一個主題」，跟既有篇的關係要事後補。

**推導式閱讀路線成立**。「完整推導」的讀法（從源頭一路展開）只在推導體系上是真路線；主題集合的「建議順序」只是排列。

## 具體 case

### case 1：同一篇文章在兩種結構下的角色

重寫前的第一篇（planning 心態篇）是主題之一：40 行講四個話題（需求預判、需求模型、五個動作、異常常態），「代價不對稱」以一句斷言的形式路過。重寫後同一個檔案被重新定位成**模組的推導源頭**：兩側成本展開、量級示意算式、從不對稱推出三個行為特徵與五條工作線的存在理由、四個失效條件——之後每一篇的「下一步」或機制段都有回到它的路。位置沒變、責任變了：從「第一個主題」變成「所有判準的基準」。

### case 2：源頭寫進模組索引

重寫後的模組 `_index` 第二段明寫「推導的起點是一個成本不對稱……模組的每一篇都從這個不對稱往外展開」。這句話是可檢查的：拿任何一篇的核心判準問「折算回源頭是什麼」，答不出來的篇章就是掛載失敗的訊號。重寫前的 `_index` 只有主題羅列，這個檢查無從做起。

## 沒這樣做的麻煩

### 深度補不上、因為沒有方向

主題集合被批評「淺」的時候，每篇能做的只有「多寫一點」——往哪裡深沒有答案。推導體系的深化方向是給定的：把這篇的判準往源頭折算得更緊。這正是 batch 裡「補深」實際的操作方式。

### 判準各自為政、審查抓不到體系級錯誤

斷言支撐 frame 能檢查單篇判準的支撐，但兩篇判準互斥這種體系級錯誤，要有共同基準才可見（機制見核心原則的「跨篇矛盾才會現形」段）。

### 讀者記規則、不是學會用尺

沒有共同基準的模組，讀者離開時帶走 N 套判準；有基準的模組，讀者帶走一把尺加 N 個用法——後者才遷移得動。

## 跟其他抽象層原則的關係

- **[#218 文章按分析弧拆分](../article-split-by-analysis-arc-not-source-spread/)**：兩個層級的結構原則。#218 管篇內縱深（一篇走完情境 / 機制 / 量化 / 判準 / 失效），本卡管模組級橫向結構（各篇的判準掛在同一個源頭上）。一個模組可以每篇都走完弧、仍是主題集合——兩個檢查互不替代。
- **[#211 複合問題先拆機制再談交互](../compound-problem-decompose-then-interact/)**：#211 說機制要各自成篇、交互用連結串接；本卡補「拆開的機制要有一個共同源頭」——拆而無源是主題集合、拆而有源是推導體系。
- **[#209 教鑑別能力、不教操作流程](../teach-judgment-not-procedure/)**：判斷力導向在模組層的展開。單篇的判斷力來自機制主線（#209）、模組的判斷力來自可折算的共同基準（本卡）——讀者面對模組沒直接覆蓋的新情境時，靠的是把源頭機制自己往外推。
- **[#216 經驗談來源要重建分析層](../anecdotal-source-needs-mechanism-reconstruction/)**：先後關係。#216 把個別斷言還原成機制，是本卡的前置——沒有機制就沒有可掛載的東西；機制重建完成後，本卡回答「這些機制怎麼組織成模組」。

## 判讀徵兆

| 徵兆                                                   | 該做的行動                                     |
| ------------------------------------------------------ | ---------------------------------------------- |
| 模組 `_index` 說不出「推導的起點是什麼」               | 主題集合訊號——找出（或建立）各篇判準的共同基準 |
| 任選一篇的核心判準、無法折算回某個共同機制             | 該篇掛載失敗——補折算路徑、或承認它屬於別的模組 |
| 想補深某篇但說不出「往哪裡深」                         | 沒有源頭給方向——先立源頭再補深                 |
| 新增篇章時只能「再加一個主題」、跟既有篇的關係事後才補 | 掛載點缺席——擴篇前先問新篇折回源頭的路徑       |
| 兩篇的判準疑似互斥、但說不清楚為什麼算矛盾             | 缺共同基準——體系級審查要先有體系               |

## 適用範圍與邊界

- **適用**：分析導向的教學模組（商業分析、架構決策、選型教材）——判準密集、讀者要帶走判斷力的內容。
- **邊界**：
  - **目錄型模組不適用**：vendor overview、工具清單、case 庫這類本來就是集合的內容，硬找源頭是強加敘事。
  - **異質 case 記錄不適用**：work-log 型內容的價值在事件本身，不需要推導體系。
  - **源頭是折算基準、不是開場模板**：每篇的判準要能折回源頭，但每篇的開場不必都從源頭講起——把源頭寫成固定開場會複製 cadence 同質化（見 [#122](../cadence-homogenization-in-batch-writing/) 的教訓）。

## Self-case：本卡的觸發來源

觸發於採購模組重寫的 retrospective。重寫前 6 篇平行主題、深度被批評後「不知道往哪補」；重寫時第一個結構決策是把成本不對稱立為源頭、第一篇改寫為推導起點，之後所有篇章的判準向它折算。收益在後續流程中逐一兌現：Round 3 兩個獨立 reviewer 收斂抓到跨篇判準矛盾（免費選擇權 vs 訂單信用）——矛盾可見的前提是兩篇同屬一個推導體系；backlog 擴篇（漲價應對、呆料處置）時掛載點明確、當天完成且跟既有篇零衝突；模組 `_index` 的「完整推導」讀法路線由此成立。
