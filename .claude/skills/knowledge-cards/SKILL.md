# Knowledge Cards（知識卡生命週期）

知識卡是內容體系的原子抽象層：一張卡承載一個術語的核心概念、讓文章用連結取代行內解釋、讓解釋只寫一次被全體引用。本 skill 標準化卡片的完整生命週期——從「這個詞該不該有卡」到「模組級的缺卡 audit」——讓建卡動作在任何模組、任何批次都走同一套判準。

---

## 生命週期六站

| 站      | 核心問題               | 關鍵判準                                                                      |
| ------- | ---------------------- | ----------------------------------------------------------------------------- |
| 1 判準  | 這個術語該有卡嗎？     | 教學需求：目標讀者群裡最不熟悉的那端能不能理解；出現頻率只是輔助訊號          |
| 2 落點  | 卡住在哪個目錄？       | 概念的主要領域歸屬；一卡一住址、跨模組用連結不重複建卡                        |
| 3 撰寫  | 卡片長什麼樣？         | 四段結構（定義 / 概念位置 / 訊號與例子 / 判讀方式）＋首段與概念位置段鄰卡連結 |
| 4 登記  | 卡片在哪些地方要現身？ | collection index 分類表＋lint 工具的作用域清單——兩者都是必經註冊點            |
| 5 回填  | 文章怎麼接上卡？       | 每篇文章該術語第一次出現處連卡、後續裸用                                      |
| 6 Audit | 存量模組的缺口在哪？   | 詞頻掃描 × 缺卡判定矩陣 → 優先序 → 批次建卡                                   |

## 缺卡判定矩陣（速查）

術語處理分三級：行內展開（正文 1-2 句定義）、連卡、裸用。判定用兩軸——**術語在本文的角色**（主線 / 支撐 / 背景）×**是否在本文的前置知識基線內**（基線＝模組讀者定位 ∪ 閱讀路線中前置篇章教過的主線術語）：

| 術語角色 | 基線外                        | 基線內                     |
| -------- | ----------------------------- | -------------------------- |
| 主線     | 行內展開＋連卡                | 行內展開（本文就是在教它） |
| 支撐     | **連卡（卡不存在 → 先建卡）** | 連卡（預設）或裸用         |
| 背景     | 連卡                          | 可裸用                     |

「支撐 × 基線外 × 無卡」就是缺卡的精確定義。文章越深、越多術語落入基線、行內解釋自然遞減；連卡密度不遞減——它是空降讀者（搜尋直達深度篇）的回路。

## 註冊點清單（漏一個都不報錯）

新卡片或新卡片目錄需要在數個彼此獨立的註冊點現身、漏掉任何一個都不產生錯誤訊號：

1. **Collection index 分類表**——目錄的索引頁要列出這張卡（實測發現過整批卡運作正常、文章連得到、只有索引層看不見它們）
2. **Lint 工具的作用域清單**——frontmatter 必填檢查跟卡片結構檢查（鄰卡連結 / orphan）通常是**兩個獨立的目錄清單**、都要登記。作用域外的目錄跑檢查會得到零 error——那是「沒被看」不是「沒問題」、檢查通過的假象比沒檢查更危險
3. **上層入口**——新建卡片目錄時、模組首頁或站台首頁的對應分類段要同步
4. **Weight 編號帶**——新卡接續該目錄的既有編號帶、不隨手給不撞號的值

## 跟其他寫作流程的分工

本 skill 管卡片的生命週期（判準 / 落點 / 登記 / 回填 / audit）。卡片內文的字句層品質（正向陳述、中性 register、地區用語）屬單篇寫作方法的範疇——專案若有對應的寫作 skill、批次建卡後的字句層自審套用該 skill 的 keyword bank；批次多輪審查的 frame 規劃屬 review 流程 skill。三者疊加、互不取代。
