# Card Authoring（建卡判準與撰寫）

本 reference 為「寫一張或一批新知識卡」情境：判斷該不該建卡、卡放哪個目錄、標題與內文怎麼寫、批次生產的紀律。

適用：新術語建卡、批次補卡、審查既有卡片是否合格。
不適用：模組級的缺卡掃描與回填規劃（見 SKILL.md 路由到 gap-audit-and-backfill）。

> **自包含聲明**：閱讀本文件不需要先讀其他 reference。

---

## 建卡判準：教學需求、不是出現頻率

一個術語值得建卡的判準是**教學需求**：目標讀者群裡最不熟悉的那端、缺了這個術語的知識就難以理解教材。出現頻率與是否影響實作判斷只是輔助訊號——一個只出現在單篇文章的術語、只要讀者缺它就讀不懂、就值得建卡。

常識是相對於讀者背景的：同一個術語對甲背景的讀者是常識、對乙背景完全陌生。跨背景讀者群的教材裡、幾乎所有領域特定術語都需要卡。建卡的邊際成本低（一張卡 40-60 行）、讀者缺卡的代價高（離開教材去搜尋、可能找到口徑不一致的解釋）。

**判斷成本測試**：讀者如果不知道這個名詞、會誤判某段分析的結論、或無法解碼一張表——就建卡。

**錯**（頻率邏輯）：「這個詞只在一篇文章出現、撐不起一張卡」。
**對**（教學邏輯）：「最不熟悉的讀者看到這個詞會卡住嗎？會、就建卡——即使全站只出現一次」。

排除項：過度寬泛的傘狀詞（「策略」「成長」「轉型」）建不出單一語意責任、在文章內直接補情境精確的說明；僅在單篇文章內成立的臨時詞同樣排除。

## 拆卡與合卡

一張卡只承擔一個語意責任。兩個判準方向：

- **拆**：名詞跨情境變義時必拆——同一個詞在 A 情境跟 B 情境指不同東西、混寫會讓引用端各取所需。
- **合**：相鄰的 sibling 概念若共享同一個語意責任、可以合為一卡——例如「有形資產的分年認列」跟「無形資產的分年認列」是同一機制的兩個面向、常一起出現、合為一卡；反例是「合併報表」跟「非控制權益」——相關但各自承擔獨立的判讀責任、拆兩張互連。

判斷句：「這張卡聲稱解決的議題、切完整了嗎？」合卡後任一面向講不完整、就拆。

## 落點：卡住在哪個目錄

- **主要領域歸屬**：誰教這個概念、誰持有卡。跨模組共用的術語住在概念的主場模組、其他模組用連結引用——一卡一住址、跨模組不重複建卡（重複建卡會產生兩個版本的定義、口徑漂移只是時間問題）。
- **新目錄門檻**：模組還沒有卡片目錄時、累積 3 張以上的缺卡再建目錄（附 collection index）；未達門檻的過渡期先行內展開。
- 新目錄建立時走 SKILL.md 的註冊點清單（index、lint 作用域、上層入口）。

## 命名：原文為權威錨點

**標題以術語原文（多為英文）為主、在地語言為輔助**：`Operating Margin（營業利益率）`。理由有兩層：術語的原文社群與學術資料遠比譯文資料豐富、原文是讀者往外查證的唯一穩定錨點；在地翻譯跨地區有出入（同一個英文術語在不同華語地區的譯名與慣用解釋不同）、以任何單一地區的譯名當權威都會讓其他地區讀者對不上。

- slug / 檔名用英文 kebab-case（`operating-margin`）
- 標題格式：`原文（在地譯名）`；縮寫已是通行錨點時可直接用（`EPS（每股盈餘）`）
- **邊界**：在地產業特有、原文世界沒有對應詞的術語（例如某地特有的產業品項）以在地語言為主、有對應原文時附上
- 集合命名不內嵌數量（「三率」這種列在內文、不烤進卡名）

## 卡片結構

四段結構、每段一個職責：

```markdown
---
title: 原文術語（中文輔助）
date: YYYY-MM-DD
description: recall trigger——什麼情境下需要回來讀這張卡
weight: 接續該目錄的既有編號帶
tags: [依目錄慣例]
---

開頭段：定義核心概念、回答「這是什麼」。公式類術語直接給公式。首段必含至少一條鄰卡連結。

## 概念位置

這個概念在推理鏈中的位置、跟鄰近概念的關係。必含至少一條鄰卡連結。

## 可觀察訊號與例子

什麼時候這個概念變成判讀重點、一到兩個具體情境（帶真實數據時標年份——數據會過期、年份讓過期可辨識）。

## 判讀方式

遇到這個概念要做什麼判斷、常見誤判是什麼。有對應深入長文時在此連結（卡答「是什麼」、長文答「怎麼判讀」）。
```

- 開頭段先定義再舉例——定義是讀者 30 秒拿走的東西
- description 寫成 recall trigger（回答「什麼情境下需要回來讀」）、四不要：不塞操作細節、不加假設前提、不用情緒修辭、不內嵌數字
- 鄰卡連結是硬要求（首段＋概念位置段各至少一條）——孤立的卡是字典條目、連進網的卡才是知識體系

## 批次生產紀律

一次寫多張卡時、cadence 同質化會發生在開場句式、段落骨架、判讀段的收尾式——單張自審全部抓不到。生成端輪替：開場句式輪換（公式起手 / 位置起手 / 機制敘事起手 / 用途起手 / 對比起手）、例子的情境輪換、判讀段的結構輪換。批次完成後連讀整批比句式骨架、再跑字句層掃描（裝飾符號 / 對讀者喊話 / 絕對化語氣——專案有寫作 skill 時套用其 keyword bank）。

**字句層命中是候選、不是判決**：掃描命中後要做語意判定再改。兩類實測過的 false positive——(1) 泛用詞出現在字面意義（「賣了東西還沒收到錢」的「東西」指商品實物）跟濫用（不同情境壓成同一模糊標籤）是兩回事、字面意義可保留、改寫成更精確的詞也無妨；(2)「必然 / 天生」描述數學或會計的結構性結果（持股 45% 則歸母佔比必為 45%）跟把設計選擇講成自然法則是兩回事、前者是真必然可斷言——判別線是「這個必然有沒有上游設計選擇當前提」。

**批次完成後的機械自驗**（結構層、跑完才算完成）：

```bash
for f in <new_card_files>; do
  intro=$(awk '/^---$/{c++;next} c==2 && /^## /{exit} c==2{print}' $f | grep -c "](<cards_route>")
  pos=$(awk '/^## <concept_position_heading>/{p=1;next} p && /^## /{exit} p{print}' $f | grep -c "](<cards_route>")
  echo "$f 首段連結:$intro 概念位置連結:$pos"   # 任一為 0 = 違規
done
```

實測教訓：一批 10 張卡、其中 1 張首稿的首段漏了鄰卡連結、逐張自讀沒抓到、跑這段統計立刻現形——鄰卡連結是硬要求、驗證要機械化、不靠印象。

## 自檢清單

- [ ] 教學需求判準通過（最不熟悉的讀者需要它）、非傘狀詞
- [ ] 一卡一語意；sibling 合卡時每個面向都講完整
- [ ] 落點正確：概念主場模組、無重複建卡
- [ ] 標題原文為主（中文輔助）、slug 英文 kebab-case
- [ ] 四段結構完整、首段與概念位置段各有鄰卡連結
- [ ] description 是 recall trigger、weight 接續編號帶
- [ ] 例子的時效數據標年份
- [ ] 批次時開場句式有輪替、整批連讀過
- [ ] 登記進 collection index 分類表（見 SKILL.md 註冊點清單）
